# 연습 문제

## 개요

301에서 배운 오케스트레이션과 워크플로우를 직접 만들어보세요.

## 연습 1: 간단한 워크플로우 만들기

### 문제

n8n으로 다음 워크플로우를 만드세요.

**시나리오:** 새로운 이메일이 오면 Slack으로 알림을 보내고, Google Sheets에 기록합니다.

### 단계

1. Gmail Trigger 설정
2. Slack 메시지 전송
3. Google Sheets에 행 추가

### 제출물

- n8n 워크플로우 스크린샷
- 실행 결과 캡처

AI Agent를 사용하여 이메일을 분류하고 요약하는 워크플로우를 만드세요.

- 프로모션

### 제출물

## 연습 3: RAG 구축

### 문제

관심 있는 주제의 문서 5개를 수집하여 간단한 RAG 시스템을 만드세요.

### 단계

1. 문서 수집
2. 벡터화
3. 질문-답변 테스트 3개

### 제출물

```
주제: [주제명]

문서 목록:
1. [제목]
2. [제목]
...

Q&A:
1. Q: [질문]
   A: [답변]

2. Q: [질문]
   A: [답변]

3. Q: [질문]
   A: [답변]
```

## 연습 4: A/B 테스트

### 문제

동일한 질문에 대해 두 가지 다른 프롬프트로 답변을 받고 비교하세요.

### 질문

"AI 도입이 필요한 회사 내 프로세스 3가지를 제안해주세요"

### 프롬프트

**프롬프트 A:** 간단한 프롬프트
**프롬프트 B:** 구조화된 프롬프트 (Context, Examples, Format 포함)

### 평가

| 항목 | 프롬프트 A | 프롬프트 B |
|------|------------|------------|
| 정확도 | /5 | /5 |
| 관련성 | /5 | /5 |
| 완결성 | /5 | /5 |
| 총점 | /15 | /15 |

### 제출물

- 두 프롬프트 내용
- 답변 비교
- 평가 결과

## 연습 5: 종합 프로젝트

### 문제

자신의 업무에서 자동화할 수 있는 프로세스를 하나 선택하여 완전한 워크플로우를 만드세요.

### 요구사항

1. 최소 3개 노드 사용
2. AI Agent 1개 이상 포함
3. 외부 API 1개 이상 연동

### 제출물

```
[프로젝트 개요]

목적:
[자동화하려는 작업과 목적]

워크플로우 다이어그램:
[노드 간 연결도]

사용한 도구:
- 노드 1: [도구명, 용도]
- 노드 2: [도구명, 용도]
- ...

실행 결과:
[Before vs After 비교]

개선점:
[추가 개선할 부분]
```
