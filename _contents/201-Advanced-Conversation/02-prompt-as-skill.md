# 2. Prompt-as-a-Skill & Prompt-as-a-Use-Case

## 개요

프롬프트 엔지니어링의 기본 공식과 스킬 저장소를 활용하는 방법을 학습합니다.

## 2.1 프롬프트 엔지니어링이란?

프롬프트 엔지니어링은 AI로 원하는 결과를 얻기 위해 입력을 설계하는 기술입니다.

## 2.2 프롬프트 공식 5가지

### 1. Context (맥락)

```
[나쁜 예]
블로그 글 써줘

[좋은 예]
당신은 10년 경력의 기술 블로거입니다.
주제: AI 도입 전략
타겟 독자: 비기술 경영진
목적: AI 도입을 설득하기 위한 글
```

### 2. Clear Instruction (명확한 지시)

```
[나쁜 예]
이거 요약해줘

[좋은 예]
다음 문서를 3문장으로 요약해주세요.
1. 핵심 문제
2. 제안된 해결책
3. 기대 효과
```

### 3. Examples (예시)

```
[나쁜 예]
이메일 답변 써줘

[좋은 예]
다음 스타일로 이메일 답변을 써주세요:

예시:
"안녕하세요 [이름],
요청하신 [주제]에 대해 답변드립니다...
..."
```

### 4. Format (형식)

```
[형식 지정 예]
다음 형식으로 답변해주세요:
## 요약
[3문장 요약]

## 핵심 포인트
1.
2.
3.

## 다음 단계
- [ ] ...
```

### 5. Constraints (제약 조건)

```
[제약 조건 예]
- 길이: 500자 이내
- 어조: 전문적이지만 친근하게
- 포함해야 할 키워드: AI, 자동화, 효율성
- 피해야 할 표현: ~인 것 같습니다, ~일 수 있습니다
```

## 2.3 스킬 저장소 활용

22,000개 이상의 프롬프트 템플릿이 공유되어 있습니다.

<!-- INFORAPHIC PLACEHOLDER: 스킬 저장소 카테고리 -->
![스킬 저장소](_assets/prompt-repository.png)
*디자인 필요: 카테고리별 스킬 저장소 구조*

### 주요 스킬 카테고리

| 카테고리 | 예시 |
|----------|------|
| 마케팅 | SNS 캠페인, 광고 카피, 이메일 마케팅 |
| 세일즈 | 콜드 이메일, 협상, 제안서 |
| 분석 | 데이터 분석, 시장 조사, 경쟁 분석 |
| 글쓰기 | 블로그, 보고서, 뉴스레터 |
| 프로덕트 | PRD, 사용자 스토리, 기능 명세 |

## 2.4 Prompt-as-a-Use-Case

프롬프트를 재사용 가능한 유스케이스로 정의하는 방법입니다.

### 유스케이스 템플릿

```markdown
# [유스케이스 이름]

## 목적
[이 유스케이스의 목적]

## 입력
- [필요한 입력 정보 1]
- [필요한 입력 정보 2]

## 프롬프트 템플릿
```
[재사용 가능한 프롬프트]
```

## 예시
[입출력 예시]

## 변형
- [변형 1]
- [변형 2]
```

- **5가지 공식**: Context, Instruction, Examples, Format, Constraints
- **스킬 저장소**: 22,000+ 프롬프트 템플릿 활용
- **유스케이스**: 재사용 가능한 프롬프트 템플릿
