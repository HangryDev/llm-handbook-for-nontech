## 4.1 Personal RAG 시스템

논문과 문서로 AI에 전문가 빙의시키는 시스템입니다.

<!-- DEMO VIDEO PLACEHOLDER -->
[DEMO VIDEO: Personal RAG 구축 실습 (4분)]

### RAG란?

Retrieval-Augmented Generation의 약자로, 개인 문서에서 정보를 검색하여 답변을 생성합니다.

### Personal RAG 구축 단계

1. **문서 수집**
   - PDF, 웹페이지, 노트 등
   - 논문, 보고서, 블로그 포스팅

2. **전처리**
   - 텍스트 추출
   - 청킹 (Chunking)
   - 메타데이터 추가

3. **벡터화**
   - 임베딩 모델로 변환
   - 벡터 데이터베이스 저장

4. **검색 및 생성**
   - 질문 → 관련 문서 검색
   - 검색된 문서 + 질문 → 답변 생성

### 도구

| 도구 | 특징 |
|------|------|
| NotebookLM | Google, 간편한 UI |
| GPTs | OpenAI, 커스터마이징 |
| ChatPDF | PDF 전문 |
| PrivateGPT | 로컬, 프라이버시 |

## 4.2 OpenClaw (Claude Desktop)

Claude Desktop으로 개인 PC를 조작하는 방법입니다.

### Claude Desktop 설정

1. **설치**
   - Claude Desktop 다운로드
   - API Key 설정

2. **MCP (Model Context Protocol)**
   - 로컬 파일 읽기
   - 시스템 명령 실행
   - 외부 도구 연결

<!-- DEMO VIDEO PLACEHOLDER -->
[DEMO VIDEO: Claude Desktop MCP 설정 (3분)]

### 사용 예시

```
사용자: "내 문서 폴더에서 지난주 업무 보고를 찾아줘"

Claude: [MCP를 통해 로컬 파일 검색]
"문서 폴더에서 다음 파일을 찾았습니다:
- 2024-03-15_업무보고.docx
- 2024-03-18_주간회의록.pdf"
```

### Claude Code 기능

- 코드 생성 및 수정
- 디버깅
- 코드 리뷰
- 문서화

### Hooks 설정

```json
// .claude/settings.json
{
  "hooks": {
    "prePrompt": "현재 프로젝트의 구조를 파악하세요",
    "postResponse": "변경된 파일을 요약해주세요"
  }

```

## 4.4 Agent 구성

복잡한 작업을 수행하는 에이전트를 만듭니다.

### Agent 구조

```
┌─────────────────────────────────┐
│         사용자 요청              │
└─────────────┬───────────────────┘
              │
┌─────────────▼───────────────────┐
│         Planner                 │
│    (작업 계획 수립)              │
└─────────────┬───────────────────┘
              │
┌─────────────▼───────────────────┐
│         Tools                   │
│  - 검색                        │
│  - 코드 실행                    │
│  - 파일 조작                    │
│  - API 호출                     │
└─────────────┬───────────────────┘
              │
┌─────────────▼───────────────────┐
│         Evaluator               │
│    (결과 평가 및 반복)           │
└─────────────────────────────────┘
```

### Agent 만들기

1. **목적 정의**
   - 에이전트가 할 일 정의

2. **도구 선택**
   - 필요한 도구 나열

3. **프롬프트 설계**
   - 시스템 프롬프트 작성
   - 도구 사용 가이드

4. **테스트**
   - 다양한 시나리오 테스트

## 4.5 재사용 가능하게 저장하기

프롬프트와 에이전트를 재사용 가능하게 저장합니다.

### 저장 방법

1. **GPTs로 저장**
   - ChatGPT 내에서 커스터마이즈
   - 공유 링크 생성

2. **Claude Projects**
   - 프로젝트 단위 관리
   - 문서 연결

3. **프롬프트 라이브러리**
   - Notion, Obsidian 등
   - 카테고리별 정리

### 프롬프트 템플릿

```markdown
# [프롬프트 이름]

## 목적
[이 프롬프트의 목적]

## 카테고리
[마케팅/세일즈/분석/등]

## 프롬프트
```
[프롬프트 내용]
```

## 사용 예시
[예시 입력]

## 태그
#태그1 #태그2
```
