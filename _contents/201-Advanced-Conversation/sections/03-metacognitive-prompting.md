# 3. Prime Instruction & 메타인지 프롬프팅

## 개요

## 3.1 작업 분해 (Task Decomposition)

복잡한 작업을 작은 단위로 나누는 기술입니다.

<!-- DEMO VIDEO PLACEHOLDER -->
[DEMO VIDEO: 작업 분해 실습 (2분)]

### 예시: 보고서 작성

```
[나쁜 예]
"연간 보고서를 작성해줘"

[좋은 예 - 작업 분해]
"연간 보고서 작성을 위해 다음 단계를 순서대로 수행해주세요:

Step 1: 보고서 구조 설계
- 목차 작성
- 각 섹션의 목적 정의

Step 2: 데이터 수집
- 재무 데이터
- 운영 데이터
- 시장 데이터

Step 3: 각 섹션 작성
- 경영진 개요
- 재무 성과
- 운영 성과
- 전망

Step 4: 검토 및 수정
- 일관성 확인
- 오탈자 검토
- 피드백 반영"
```

## 3.2 Meta Prompting

AI에게 "어떻게 생각할지"를 알려주는 기법입니다.

### Meta Prompt 프레임워크

```
당신은 [역할]입니다.

[작업 목표]

다음 사고 과정을 따라 답변해주세요:
1. [첫 번째 고려사항]
2. [두 번째 고려사항]
3. [세 번째 고려사항]

출력 형식:
[원하는 형식]
```

### 예시

```
당신은 전략 컨설턴트입니다.

우리 회사의 신규 시장 진입을 검토하고 있습니다.

다음 사고 과정을 따라 분석해주세요:
1. 시장 규모와 성장성 평가
2. 경쟁 환경 분석
3. 진입 장애물 파악
4. 우리 회사의 강점과 약점 분석
5. 진입 전략 제안

각 단계에서:
- 데이터 기반 근거 제시
- 위험 요소 식별
- 현실적인 제안 제시
```

## 3.3 Ask for Context

추가 맥락을 요청하여 더 나은 답변을 얻는 기법입니다.

### Context 질문 예시

```
최적의 답변을 위해 다음 정보가 필요합니다:
1. 타겟 고객은 누구인가요?
2. 예산 범위는 어느 정도인가요?
3. 시간 제약이 있나요?
4. 우선순위는 무엇인가요?
5. 선호하는 스타일이 있나요?

이 정보를 알려주시면 맞춤형 답변을 드리겠습니다.
```

---

## 3.4 논리 결함 파악

AI에게 자신의 답변을 비판하게 하는 기법입니다.

### 예시

```
다음 답변에 대해 비판적인 검토를 해주세요:

[답변 내용]

검토 항목:
1. 논리적 오류가 있는가?
2. 가정은 타당한가?
3. 반대 증거는 무시되지 않았는가?
4. 결론은 근거를 충분히 지지하는가?

비판 후, 개선된 답변을 제시해주세요.
```

---

## 3.5 전문가 소환

특정 분야 전문가의 관점에서 답변을 요청하는 기법입니다.

### 예시

```
이 문제에 대해 다음 전문가 3명의 관점에서 답변해주세요:

1. 경영학 교수의 관점
   - 이론적 배경
   - 학계의 일반적 견해

2. 실무 경영자의 관점
   - 실제 경험
   - 실현 가능성

3. 컨설턴트의 관점
   - 벤치마크 사례
   - 실행 전략

각 관점의 합의점과 차이점을 정리해주세요.
```

## 3.6 AI as a Judge & 배심원단

여러 AI 응답을 비교 평가하는 기법입니다.

<!-- INFORAPHIC PLACEHOLDER: AI 배심원단 시스템 -->
![AI 배심원단](_assets/ai-jury.png)
*디자인 필요: 여러 AI가 답변하고 판사 AI가 평가하는 구조*

### 예시

```
다음 문제에 대해 3가지 다른 답변을 생성해주세요:

[문제]

각 답변에 대해 다음 기준으로 평가해주세요:
- 논리성 (1-10점)
- 창의성 (1-10점)
- 실행 가능성 (1-10점)
- 명확성 (1-10점)

최종적으로 가장 좋은 답변을 선택하고 이유를 설명해주세요.
```

## 3.7 맥락 배치하기

충분한 맥락을 제공하여 AI가 더 잘 이해하도록 하는 기법입니다.

### 맥락 배치 구조

```
# 배경
[프로젝트/문제의 배경]

# 현재 상황
[현재 어떤 상태인지]

# 제약 조건
[시간, 예산, 리소스 등]

# 목표
[달성하고 싶은 것]

# 우리가 시도한 것
[이미 시도해본 방법과 결과]

# 원하는 답변
[구체적으로 원하는 것]
```
