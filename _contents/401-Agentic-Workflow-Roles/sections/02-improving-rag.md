## 2.2 전처리 단계

### 1. 텍스트 추출

- PDF → 텍스트
- 이미지 → OCR
- 웹페이지 → 텍스트

<!-- DEMO VIDEO PLACEHOLDER -->
[DEMO VIDEO: 텍스트 추출 실습 (2분)]

### 2. 클리닝

- 불필요한 문자 제거
- 불용어 제거
- 형식 정규화

### 3. 청킹 (Chunking)

문서를 적절한 크기로 나눕니다.

**청킹 전략:**

| 전략 | 설명 | 장점 | 단점 |
|------|------|------|------|
| 고정 크기 | 일정한 토큰 수로 분할 | 간단 | 맥락 끊김 |
| 문단 단위 | 문단 기준 분할 | 맥락 유지 | 크기 불균형 |
| 의미 단위 | 의미 완결성 기준 | 최적 맥락 | 복잡함 |

### 4. 메타데이터 추가

각 청크에 메타데이터를 추가합니다.

**메타데이터 예시:**

```json
{
  "content": "청크 내용...",
  "metadata": {
    "source": "보고서_2024.pdf",
    "page": 15,
    "chapter": "재무 현황",
    "date": "2024-03-01",
    "author": "재무팀",
    "tags": ["재무", "2024", "실적"]
  }
}
```

---

## 2.3 메타데이터 디자인

### 좋은 메타데이터의 조건

1. **검색 가능성**
   - 자주 검색할 키워드 포함

2. **필터링 가능성**
   - 범위로 좁힐 수 있어야 함

3. **일관성**
   - 통일된 형식

### 메타데이터 스키마 예시

| 필드 | 타입 | 예시 | 설명 |
|------|------|------|------|
| source | string | "보고서_2024.pdf" | 출처 문서 |
| page | number | 15 | 페이지 번호 |
| chapter | string | "재무 현황" | 챕터명 |
| date | date | "2024-03-01" | 작성일 |
| author | string | "재무팀" | 작성자 |
| tags | array | ["재무", "2024"] | 태그 |

---

## 2.4 검색 쿼리 개선

### 메타데이터 필터링

```python
# 검색 시 메타데이터 필터 적용
results = vector_store.search(
    query="2024년 매출 현황",
    filter={
        "date": {">=": "2024-01-01"},
        "tags": {"in": ["재무", "실적"]}
    }
)
```

### 하이브리드 검색

- 벡터 검색 + 키워드 검색
- 더 정확한 결과

## 요약

- **전처리**: 추출 → 클리닝 → 청킹 → 메타데이터
- **청킹 전략**: 고정 크기, 문단 단위, 의미 단위
- **메타데이터**: source, page, chapter, date, author, tags
- **검색 개선**: 필터링, 하이브리드 검색
- **Reranking**: 결과 재정렬
