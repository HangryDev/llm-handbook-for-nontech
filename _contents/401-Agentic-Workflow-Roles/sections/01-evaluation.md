## 1.2 Designing Your Eval Workflow

### 평가 워크플로우 설계

1. **목표 정의**
   - 무엇을 평가할 것인가?

2. **지표 선택**
   - 어떤 메트릭을 사용할 것인가?

3. **데이터 준비**
   - 테스트 케이스 준비

4. **실행**
   - 자동/수동 평가

5. **분석**
   - 결과 분석 및 개선

---

## 1.3 지표 (Metrics)

### 주요 평가 지표

| 지표 | 설명 | 측정 방법 |
|------|------|-----------|
| 정확도 | 올바른 답변 비율 | 정답과 비교 |
| 관련성 | 질문과의 관련성 | LLM as Judge |
| 완결성 | 답변의 완결성 | 체크리스트 |
| 일관성 | 반복 실행 시 일관성 | 여러 번 실행 |
| 속도 | 응답 시간 | 시간 측정 |
| 비용 | API 호출 비용 | 토큰 수 |

### 정성적 vs 정량적 지표

**정량적:**
- 정확도, 속도, 비용

**정성적:**
- 관련성, 완결성, 유용성

---

## 요약

- **EDD**: 평가 주도 개발
- **평가 워크플로우**: 목표 → 지표 → 데이터 → 실행 → 분석 → 개선
- **지표**: 정확도, 관련성, 완결성, 일관성, 속도, 비용
